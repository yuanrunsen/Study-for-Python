在list 中 查找时间会随着数据的增加而增加很大

在dict中 查找不会随着数据的量增大而增大，并且查询时间很小很小

例子：
a=['e','d','c','b','a']
b={'a':1,'b':2,'c':3}
if value in a:
和if value in b:

注意 dict的默认迭代是key

dict和set的底层是哈希表来实现，所以查询的时候就不用像列表一样遍历，直接取即可
dict会分配空白的表元 即多分配内存，内存花销比较大

dict取值或查找的时候 先计算key的哈希值 来定位表中的一个表元 如果表元为空输出keyerror
如果不为空 先判断是否冲突（即判断两边的key是否相等 如果不相等 接着解决冲突）

表元里存的是key和value
